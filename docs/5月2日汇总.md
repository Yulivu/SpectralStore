## 2. Solid 的好消息

### 2.1 系统已经跑通

Exp3（查询中心 benchmark：在压缩域端到端测试 Q1-Q5，想证明 SpectralStore 不只是压缩器，而是能直接回答查询的系统）证明当前系统能端到端回答 Q1-Q5。

代表性数据：

| 指标                                             |                    结果 | 说明                          |
| ---------------------------------------------- | --------------------: | --------------------------- |
| `spectralstore_thinking` compressed/raw sparse |                0.0689 | 保留 residual 后仍低于 raw sparse |
| Q1 RMSE factor-only                            |                0.2914 | 低秩层可回答边概率                   |
| Q1 RMSE factor+residual                        |                0.2611 | 相对改善约 10.4%                 |
| Q4 RMSE factor-only                            |                0.3209 | 低秩层可回答趋势                    |
| Q4 RMSE factor+residual                        |                0.2531 | 相对改善约 21.1%                 |
| Q5 precision/recall/F1                         | 0.665 / 1.000 / 0.799 | residual 可直接支撑异常检测          |
| `sym_svd/direct_svd` Q5 F1                     |                     0 | 无 residual 层，不能回答该异常查询      |

这个结果支持系统主线（也就是thinking里的想法）：

> SpectralStore 的价值不只是重构图，而是在压缩域组织多种查询路径。

### 2.4 关于storage gate&#x20;

storage gate 是上周发现某些压缩比大于1之后（系统为了追求训练重构误差，把 residual 存成一个比原图还大的误差表，简单地说存太多残差，而且就算只保留2%，因为空间是T × n × n，存下来的也很多），重构代码并且尝试在系统实现中加入的存储预算闸门的一个想法。思路就是至少压缩比不能大于1，由于 robust residual 可能保存大量 `(t,u,v,value)` 项，如果 residual 过密，压缩存储会反而大于 raw sparse。storage gate 的作用就是在压缩器输出 store 之后计算 `compressed_vs_raw_sparse_ratio`，如果超过预算，就按策略执行 `drop_residual`、`raise` 或只记录 diagnostic。

Exp2（Bitcoin 压缩精度/存储比实验：比较 factor-only、factor+residual 和 storage gate，想证明压缩表示在精度提升和存储预算之间的 trade-off）说明这个机制是必要的：residual 如果不受控制，会破坏压缩目标。

代表性数据：

| 设置                                                  |                结果 |
| --------------------------------------------------- | ----------------: |
| 不 drop residual, rank=2 压缩比                         | 9.13 倍 raw sparse |
| 不 drop residual, rank=20 压缩比                        | 9.73 倍 raw sparse |
| 启用 storage gate 后 thinking 压缩比                      |     约 0.12 到 0.72 |
| rank=2 训练 Frobenius factor-only -> factor+residual  |  0.9952 -> 0.0177 |
| rank=2 held-out RMSE factor-only -> factor+residual |  0.3550 -> 0.3553 |

大致表明两件事：

- residual 很强：rank=2 时 Frobenius 从 0.9952 降到 0.0177。
- 从系统存储和泛化看，residual 不能无条件保留：不 drop 时压缩比会达到 raw sparse 的 9 倍以上，而 held-out RMSE 从 0.3550 到 0.3553，基本没有改善。

所以 storage gate 的意义是把 residual 从“无限制修正项”改成“预算内事件层”。这也解释了为什么我们后面更倾向于把 residual 叫 event/anomaly layer，而不是 clean correction。

可支持的新说法：

> residual 必须被预算控制；它不是越多越好，而是系统规划器在预算内选择是否保留和访问的事件层。

### 2.3 ARD 的 bug 已修，exact low-rank 小闭环完成了

Exp5 / ARD diagnostic（自动秩选择诊断实验：比较 ARD、fixed exact rank 和 over-rank，想证明 ARD 能否替代人工固定秩或网格搜索）说明：ARD 不能做主贡献，但修复不是没价值。现在要把它拆成两层看：第一层是机制是否能在干净低秩数据上工作，第二层是它能否在 observed graph 上稳定恢复“真实秩”。

代表性数据：

| 数据                         | true rank | ARD selected rank | accurate? | ARD Frobenius | fixed\_exact Frobenius |
| -------------------------- | --------: | ----------------: | --------: | ------------: | ---------------------: |
| exact low-rank, noise=0    |         3 |                 3 |         1 |       0.05455 |                0.05455 |
| exact low-rank, noise=0.02 |         3 |                 3 |         1 |       0.05658 |                0.05658 |
| exact low-rank, noise=0    |         5 |                 5 |         1 |       0.07247 |                0.07247 |
| exact low-rank, noise=0.02 |         5 |                 5 |         1 |       0.07341 |                0.07341 |

这说明修复后的 ARD 在 exact low-rank 小闭环里是成立的：它能选回 rank 3/5，而且 Frobenius 几乎等于 fixed\_exact。这证明之前 ARD 的尺度污染问题已经被修掉，至少机制没有完全失效。

但 observed SBM 上结果相反：

| 数据           | target rank | ARD selected rank | accurate? | ARD Frobenius | fixed\_exact Frobenius |
| ------------ | ----------: | ----------------: | --------: | ------------: | ---------------------: |
| observed SBM |           3 |                12 |         0 |        0.9070 |                 0.9355 |
| observed SBM |           5 |                12 |         0 |        0.9070 |                 0.9146 |

这里 ARD 选满 rank 12，并且 Frobenius 还略低于 fixed\_exact。如果目标是拟合 observed Bernoulli 采样图，更多 rank 确实可能拟合采样噪声，从而降低重构误差。但这也意味着 ARD 没有恢复我们直觉中的 community rank。

因此 ARD 当前暂时的结论是：

- 作为 exact low-rank 机制诊断，它是 positive的。
- 作为真实图/observed SBM 的自动秩选择主贡献，依旧不可靠。
- 可能主线实验需要继续用 fixed rank，ARD 后面再看如何处理，应该摆在什么位置上。

简单地说：ARD 机制在干净低秩小闭环里是可工作的，但 observed graph 上的“有效秩”不能简单等同于 community count

## 3. 不好的结果和风险

### 3.1 asym / 非对称机制没有普遍优于 `sym_svd`

Exp1（理论/机制验证实验：在 SBM、Spiked 等合成数据上比较 entrywise error、Frobenius error 和 runtime，想证明 asym 非对称机制是否真的优于 `sym_svd`）的代表性结果：

| 设置                               | `spectralstore` | `sym_svd` | 结论                    |
| -------------------------------- | --------------: | --------: | --------------------- |
| SBM, n=1000, T=20, Frobenius     |          0.1121 |    0.0962 | `sym_svd` 更好          |
| SBM, n=1000, T=20, max entrywise |          0.2540 |    0.2482 | `sym_svd` 略好          |
| SBM, n=2000, T=20, Frobenius     |          0.0918 |    0.0825 | `sym_svd` 更好          |
| SBM, n=2000, T=20, runtime       |          47.39s |     2.42s | `spectralstore` 慢很多   |
| Spiked, SNR=1, max entrywise     |          0.3939 |    0.4544 | `spectralstore` 有局部优势 |
| Spiked, SNR=1, Frobenius         |          0.6821 |    0.4121 | 但整体仍 `sym_svd` 更好     |

这说明不能再把 asym 写成和thinking里想的：非对称机制天然更强，可以考虑的说法是asym 可能对特定结构有价值，但当前普通 SBM 和整体重构指标不支持普适优势。

### 3.2 residual 不是 clean correction

Exp4\_v2（residual-query robustness 实验：看 residual 是否命中攻击边，以及 Q1/Q4/Q5 在攻击场景下应该如何使用 residual，想证明 residual 到底是 clean correction 还是 event/anomaly layer）显示，如果 truth 是 clean graph，factor+residual 反而会变差。

代表性数据：

| 设置                                    | factor-only Q1 RMSE | factor+residual Q1 RMSE | 说明                           |
| ------------------------------------- | ------------------: | ----------------------: | ---------------------------- |
| `sparse_spike`, eps=0.002             |              0.0128 |                  4.0076 | residual 加回了 spike 事件        |
| `block_sparse_spike`, eps=0.002       |              0.0127 |                  4.0101 | 同样远离 clean truth             |
| `targeted_cross_community`, eps=0.002 |              0.0140 |                  0.2910 | 加 residual 更像 attacked graph |

<br />

| 查询类型                    | truth                   | 应该用哪条路径            |
| ----------------------- | ----------------------- | ------------------ |
| Normal/background query | clean expected graph    | factor-only        |
| Observed/event query    | observed attacked graph | factor+residual    |
| Anomaly query           | attack edges            | residual-only / Q5 |

没法证明thinking里residual 修正查询，让结果更接近真实干净图这个想法了

### 3.3 residual 对 targeted\_cross\_community 很弱

Exp4\_v2（residual-query robustness 实验：这里重点看 targeted\_cross\_community 这类普通权重、结构性攻击是否能被 residual 捕捉）对普通权重的结构性 targeted attack 效果差。

代表性数据：

| attack kind                | 平均 precision | 平均 recall |   平均 F1 |
| -------------------------- | -----------: | --------: | ------: |
| 三类 spike                   |      约 0.750 |   约 0.725 | 约 0.628 |
| `targeted_cross_community` |      约 0.097 |   约 0.086 | 约 0.077 |

这说明 residual 当前更像“大幅值异常检测器”，而不是“所有攻击检测器”。

这会影响 robustness 叙事，可以说 residual 对 spike/event 类异常有效，不能说它对所有 adversarial attack 都鲁棒。

### 3.4 Q2/Q3 和索引层还不足

Exp3（查询中心 benchmark：这里重点看 Q2 top-k 邻居、Q3 社区查询和索引路径是否能形成系统贡献）里 Q2 和 Q3 暂时不能作为强贡献。

代表性数据：

| 指标                              |      结果 | 问题          |
| ------------------------------- | ------: | ----------- |
| Q2 recall\@k factor-only        |  0.0603 | 召回太低        |
| Q2 recall\@k factor+residual    |  0.1657 | 有提升但仍弱      |
| Q2 exact index 加速 factor-only   | 约 1.36x | 加速不够决定性     |
| Q2 exact index 加速 residual path | 约 1.57x | 仍只是原型       |
| Q3 NMI                          |     1.0 | 数据太简单，没有区分度 |

因此 PQ/MIPS、时间索引、社区倒排索引还不能写成完成贡献。它们是后续系统增强。

### 3.5 ARD 不能作为主贡献

ARD diagnostic（自动秩选择诊断实验：这里重点看 ARD 在 observed SBM 这种采样图上能否恢复有效秩）的坏消息在 observed SBM。

| 数据           | target rank | ARD selected rank | accurate? |
| ------------ | ----------: | ----------------: | --------: |
| observed SBM |           3 |                12 |         0 |
| observed SBM |           5 |                12 |         0 |

原因是 observed SBM 的 Bernoulli 采样图不是干净概率矩阵，更多 rank 会拟合采样噪声。因此 ARD 不能简单用 community count 当真实 rank 来验证。

### 3.6 大规模 sparse-native 还没完成

当前 `spectralstore_thinking` 仍主要依赖 dense stack。对于 50K+ 节点或更大真实图，内存和 SVD 成本会成为很麻烦的限制。

因此 Exp6 级别 scalability 还需要等下一步

## 4. asym 的问题

### 4.1 asym 原本的设想

原 Thinking 中 asym 的动机是（按我理解）：

- 时序图可能是有向的；
- sender 和 receiver 角色可能不同；
- 简单对称化可能丢失方向信息；
- 不同时间片可能有异方差噪声；
- mode-3 展开希望利用时间维度提高估计稳定性。

所以原本预期是：

> asym construction + mode-3 tensor basis 应该比 `sym_svd` 更好，尤其在 entrywise error 上更好。

### 4.2 当前实现已经把机制做出来了

当前 `spectralstore_thinking` 已经不是之前的原始版本：

- mode-3 unfold 已修复；
- triangular stitching 已接入；
- asym + ARD + robust 在统一 loop 中迭代；
- entrywise bound 元数据也已填充。

所以目前 asym 弱，不应简单归因于“实现完全没按 Thinking 做”。更准确是：实现已经对齐了主要机制，但机制在当前数据和指标下没有展现出预期优势。

### 4.3 在 SBM 上 `sym_svd` 很强

普通 SBM 对 `sym_svd` 特别友好：

- SBM 的期望邻接矩阵本身接近块常数结构；
- 如果图近似无向或块结构主导，左右角色差异并不强；
- 对称 SVD 正好能捕捉这种社区块结构；
- Frobenius error 更偏向整体平均误差，社区块结构会让 `sym_svd` 很占便宜。

也就是说，SBM 不是 asym 的自然优势场景。更像是问在一个主要由对称社区块决定的图里，非对称机制能不能打赢专门适合块结构的 SVD？目前的结论是不能。

这不必然推翻 asym，但没有办法说asym 普适更强。

### 4.4 triangular stitching 可能引入额外方差

asym triangular stitching 的逻辑是用不同时间子集估计上三角和下三角结构。

潜在问题：

- 时间片被拆成两部分后，每一半可用样本减少；
- 上三角和下三角来自不同时间均值，估计噪声可能不一致；
- 如果真实图没有强方向性，这种拆分不会带来信息增益，反而可能增加方差；
- 对 Frobenius 这类整体误差指标，额外方差会直接伤害表现。

所以在普通 SBM 里 asym 不如 `sym_svd`。

### 4.5 可能的原因，关于mode-3&#x20;

mode-3 unfold目前的做法本质上仍然是 batch SVD 型分解，并没有显式建模时间自相关，非平稳动态图过程等等

所以如果数据只是重复采样的 SBM 快照，mode-3 的收益可能有限。

### 4.6 asym 可能真正适用的结构还没被系统测试

当前缺少专门为 asym 设计的结构化 synthetic benchmark。

更应该测试的是：

| 结构                             | 为什么可能对 asym 有利                     |
| ------------------------------ | ---------------------------------- |
| sender/receiver role 不同        | 左右因子真的不同，对称化会丢信息                   |
| 时间角色切换                         | 节点从 sender 变 receiver，静态对称结构难表达    |
| directed burst                 | 局部有向突发事件更适合非对称表示                   |
| heteroskedastic directed noise | 不同节点/方向噪声不同，entrywise bound 可能更有意义 |
| 非互惠边结构                         | `u->v` 和 `v->u` 不同，sym\_svd 会混合方向  |

如果图确实有方向角色和非对称噪声，它应该更好；如果图是普通 SBM，它不应该被期待更好。

<br />

综上所述目前不能把 asym 单独作为主贡献

一些想法：

- 在系统中作为一个 compressor option；
- 在特定有向/异方差/角色切换结构中验证优势；
- 如果后续结构化实验成立，再另寻出路
